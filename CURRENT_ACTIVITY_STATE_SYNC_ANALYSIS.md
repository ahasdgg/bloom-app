# 🔴 КРИТИЧЕСКИЙ АНАЛИЗ: Синхронизация состояния `currentActivity`

## ВЫЯВЛЕННЫЕ ПРОБЛЕМЫ

### 1. **Отсутствие `currentActivity` в глобальном состоянии**
**Статус**: 🔴 КРИТИЧНО

**Проблема**:
- Store содержит `currentRecommendation`, но НЕ содержит `currentActivity`
- `currentActivity` должна быть отдельной сущностью, отличной от `activeActivities[]`
- Нет единого источника истины для текущей активности пользователя

**Последствия**:
- Когда пользователь переходит в меню, нет способа узнать, какая активность была активна
- Стейл-дата при навигации между экранами
- Невозможно синхронизировать состояние при возврате на главный экран

**Текущее состояние**:
```typescript
// ❌ НЕПРАВИЛЬНО - нет currentActivity
export interface AppState {
  currentRecommendation: Recommendation | null  // ← Это рекомендация, не активность
  activeActivities: ActiveActivity[]             // ← Это список, не текущая
  // ... нет currentActivity
}
```

---

### 2. **Отсутствие реактивности при навигации**
**Статус**: 🔴 КРИТИЧНО

**Проблема**:
- Navigation компонент НЕ подписан на изменения `activeActivities`
- Меню не знает о статусе текущей активности
- Нет синхронизации при переходе между экранами

**Текущее состояние**:
```typescript
// ❌ НЕПРАВИЛЬНО - Navigation не реактивна
const Navigation: React.FC = () => {
  const location = useLocation()  // ← Только маршрут, не состояние
  // ... нет useAppStore()
}
```

**Последствия**:
- Иконка 🔴 (Active) не обновляется в реальном времени
- Пользователь не видит, что активность запущена, пока не откроет экран Active

---

### 3. **Неправильная логика автозавершения**
**Статус**: 🟡 СЕРЬЁЗНО

**Проблема**:
```typescript
// ❌ НЕПРАВИЛЬНО - логика в useEffect
useEffect(() => {
  activeActivities.forEach(activity => {
    if (activity.status === 'completed') {  // ← Проверяем уже завершённую
      const elapsedMs = Date.now() - activity.startedAt
      const elapsedMinutes = Math.floor(elapsedMs / 1000 / 60)
      if (elapsedMinutes >= activity.duration) {
        handleAutoComplete(activity)  // ← Вызываем после завершения
      }
    }
  })
}, [activeActivities])
```

**Проблемы**:
- Проверяем `status === 'completed'`, но затем вызываем `handleAutoComplete()`
- Это означает, что `handleAutoComplete()` вызывается ПОСЛЕ того, как статус уже изменён
- Логика перевёрнута: должны проверять `status === 'active'`, а не `'completed'`

---

### 4. **Отсутствие синхронизации при входе в меню**
**Статус**: 🔴 КРИТИЧНО

**Проблема**:
- Нет `useEffect` в Navigation для синхронизации при фокусе
- Нет обработчика для события "пользователь вернулся в меню"
- `syncActiveActivities()` в ActiveScreen не помогает, когда пользователь на другом экране

**Текущее состояние**:
```typescript
// ❌ НЕПРАВИЛЬНО - нет синхронизации при навигации
export default function ActiveScreen() {
  useEffect(() => {
    store.syncActiveActivities()  // ← Только в ActiveScreen
  }, [store])
}
```

---

### 5. **Отсутствие "жёсткого сброса" при завершении**
**Статус**: 🟡 СЕРЬЁЗНО

**Проблема**:
- `handleAutoComplete()` устанавливает `currentRecommendation`, но не очищает `currentActivity`
- После завершения активность остаётся в памяти
- Нет явного сброса в `null` или `IDLE`

**Текущее состояние**:
```typescript
// ❌ НЕПРАВИЛЬНО - нет полного сброса
const handleAutoComplete = useCallback((activity: ActiveActivity) => {
  store.setCurrentRecommendation({...})  // ← Устанавливаем рекомендацию
  store.cancelActivity(activity.id)      // ← Удаляем из списка
  navigate('/completion')                // ← Но нет явного сброса currentActivity
}, [store, navigate])
```

---

### 6. **Утечка памяти в таймерах**
**Статус**: 🟡 СЕРЬЁЗНО

**Проблема**:
- Интервал `checkExpiredActivities` запускается каждую секунду
- Если пользователь откроет ActiveScreen несколько раз, будет несколько интервалов
- Нет проверки на дублирование интервалов

**Текущее состояние**:
```typescript
// ⚠️ ПОТЕНЦИАЛЬНАЯ УТЕЧКА
useEffect(() => {
  const interval = setInterval(() => {
    store.checkExpiredActivities()
  }, 1000)
  return () => clearInterval(interval)  // ← Очищается, но может быть несколько
}, [store])
```

---

### 7. **Отсутствие глобального таймера**
**Статус**: 🔴 КРИТИЧНО

**Проблема**:
- `checkExpiredActivities()` вызывается только в ActiveScreen
- Если пользователь на другом экране, таймер не работает
- Активность может истечь, но пользователь не узнает об этом

**Последствия**:
- Пользователь видит "30 минут осталось", переходит в меню, возвращается через час
- Активность должна была завершиться, но UI не обновился

---

## АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ

### Проблема A: Разделение ответственности
- `currentRecommendation` = рекомендация (что делать)
- `activeActivities[]` = список активных (что запущено)
- **Отсутствует**: `currentActivity` = текущая активность (что делает пользователь СЕЙЧАС)

### Проблема B: Отсутствие глобального жизненного цикла
- Нет единого места, где управляется жизненный цикл активности
- Логика разбросана по разным компонентам
- Нет централизованного обработчика событий

### Проблема C: Отсутствие реактивности в меню
- Navigation не знает о состоянии активности
- Иконка 🔴 не обновляется
- Нет визуального индикатора "активность запущена"

## СЦЕНАРИИ ОТКАЗА

### Сценарий 1: Стейл-дата при навигации
1. Пользователь запускает активность (Mountain Hiking, 180 мин)
2. Переходит в меню (🗺️ Path)
3. Возвращается в 🔴 Active
4. **ОЖИДАЕТСЯ**: Таймер продолжает отсчёт
5. **ПОЛУЧАЕТСЯ**: Таймер сбросился или показывает неправильное время

### Сценарий 2: Активность истекает, но UI не обновляется
1. Пользователь запускает активность (30 мин)
2. Переходит в меню
3. Ждёт 30+ минут
4. Возвращается в 🔴 Active
5. **ОЖИДАЕТСЯ**: Экран показывает "Time's up! Complete to finish."
6. **ПОЛУЧАЕТСЯ**: Таймер всё ещё считает, как будто активность не истекла

### Сценарий 3: Завершение активности не очищает состояние
1. Пользователь завершает активность
2. Видит экран CompletionScreen
3. Нажимает "View Your Garden"
4. Возвращается в 🔴 Active
5. **ОЖИДАЕТСЯ**: Экран пуст (активность завершена)
6. **ПОЛУЧАЕТСЯ**: Активность всё ещё видна в списке

### Сценарий 4: Несколько активностей конфликтуют
1. Пользователь запускает Hiking (180 мин)
2. Нажимает "Another" и запускает Yoga (30 мин)
3. Yoga истекает первой
4. **ОЖИДАЕТСЯ**: Yoga завершается, Hiking продолжается
5. **ПОЛУЧАЕТСЯ**: Обе активности завершаются или конфликтуют
